### 🔤 String
A string is a collection of characters written inside quotes.

Examples:
- "Hello"
- "Patna"
- "123"


### 📦 Empty String vs Space String

Empty String:
- "" 
- String
- Length = 0

Space String:
- " "
- String
- Length = 1
- Space is also a character


### 🏷️ Literal vs Identifier

String Literal:
- "Patna"

Identifier:
- Patna

- Quotes → String Literal
- No Quotes → Identifier (Name)


### ❌ SyntaxError vs ⚠️ NameError

SyntaxError:
- name = "Nainu
- Grammar is wrong
- Execution never starts

NameError:
- name = Nainu
- Grammar is correct
- Python searches memory
- Name not found


### 🧠 Parser vs Execution

Parser:
- Checks grammar
- Checks structure

Execution:
- Runs code
- Searches memory
- Uses values


### 🔗 Object vs Reference

- city = "Patna"
- `city` → Reference
- `"Patna"` → Object

Concept:
```text
city ───► "Patna"
```


## ⭐ Key Learning
- Strings are written inside quotes.
- Empty string and space string are different.
- Quotes create string literals.
- Without quotes, Python treats text as a name.
- Parser checks grammar.
- Execution checks memory.
- SyntaxError and NameError are different.
- Variables are references to objects.
- Parser and execution examples
- Parser checks grammar.
- Execution checks memory.
- These are two different stages in Python
